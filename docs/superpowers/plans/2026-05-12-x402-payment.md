# x402 v1 Payment Implementation Plan

**Goal:** Pay Cloudup uploads via x402 v1 (EIP-3009 `transferWithAuthorization`) so the wallet only needs USDC — no ETH for gas. The server submits the meta-transaction on the user's behalf.

**Flow:**
1. Cloudup `tools/call quick_upload` returns x402 challenge inline (success response, content text part holds `{x402Version, accepts: [...]}`)
2. We sign an EIP-712 `TransferWithAuthorization` for the requested asset/amount/recipient
3. Retry the tool call with `X-PAYMENT: <base64(payload)>` HTTP header
4. Server validates, settles on-chain, returns share URL

## Tasks

### 1. EIP-3009 typed-data hashing
- New: `Sources/CloudupSnapCore/Wallet/EIP3009.swift`
- One function: `transferWithAuthorizationDigest(...) -> Data (32 bytes)`
- Inputs: domainName, domainVersion, chainId, verifyingContract, from, to, value (UInt256 big-endian bytes), validAfter, validBefore, nonce
- Hardcoded TransferWithAuthorization type — no generic EIP-712 (we deleted that with the mpp-remote pivot; can re-add later if needed)

### 2. X402 payload models
- New: `Sources/CloudupSnapCore/Payment/X402.swift`
- `X402PaymentRequirements` (parsed from challenge `accepts[0]`)
- `X402Authorization` + `X402PaymentPayload` (the thing we base64-encode)
- `parseX402Challenge(text:) -> X402PaymentRequirements?` (move from Uploader)

### 3. Wallet sign helper
- Extend `Wallet` (and `WalletProtocol`) with `signX402Payment(_ req: X402PaymentRequirements) throws -> X402PaymentPayload`
- Generates random 32-byte nonce, computes digest, signs via Secp256k1Signer, packs result

### 4. Uploader retry path
- On x402 challenge detection: call `payment.settleX402(requirements)` → returns base64 header value
- Re-call `mcp.callTool(... extraHeaders: ["X-PAYMENT": value])`
- Same single-retry semantics as the existing mpp-remote path

### 5. PaymentClient extension
- Add `settleX402(_ req: X402PaymentRequirements) throws -> String` (base64 of payload)
- Reuse `capUSD` check before signing

### 6. Test
- Unit: known-vector test for `transferWithAuthorizationDigest` (use Hardhat test key #0, simple inputs, cross-check with viem)
- Integration: real upload against live Cloudup stage (gated)

### 7. Cleanup
- Keep `PaymentError.paymentRequired` as a fallback for unrecognized challenges
- Update FundingPanel copy to remove "ETH for gas" mention if we use only x402 going forward — or keep it since the old mpp-remote path still works for non-x402 servers
